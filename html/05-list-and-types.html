Listas HTML

Las listas HTML son elementos estructurales que se utilizan para agrupar conjuntos de información relacionados. Al usar etiquetas de lista esrabdarizadas, se mejora la accesibilidad del documentom se permite que los lectores de pantalla interpreten las relaciones entre el contenido y se proporciona una organización visual clara para los lectores, HTML ofrece tres tipos principales de listas, cada uno con una función semántica distinta.

Listas no ordenadas (<ul>)

La lista desordenada se utiliza cuando la secuencia de los elementos no es relevante. Por defecto, los navegadores las muestrab con viñetas. la <ul> etiqueta actúa como contenedor, mientras que los elementos individuales se envuelven en <li> etiquetas (elemento de lista). 

            
              <ul>
                <li>First item</li>
                <li>Second item</li>
                <li>Third item</li>
              </ul>
  • First item
  • Second item
  • Third item

Lista ordenadas (<ol>)

Las listas ordenadas se utilizan para contenido donde el orden importa, como los pasos de una receta o una lista de elementos clasificados. Los navegadores numeran automáticamente estos elemetos.

Puedes controlar la numeración mediante atributos:

  • type: Cambia el marcador (por ejemplo, '1', 'A', 'a', 'I', 'i').
  • start: Especifica el número entero inicial para la lista.
  • reversed: Invierte el orden de numeración. 
            
              <ol type="1" start="3">
                <li>Initialize project</li>
                <li>Configure settings</li>
                <li>Launch application</li>
              </ol>
  1. Initialize project
  2. Configure settings
  3. Launch application

Listas de descripción (<dl>)

La lista de descripciones está diseñada específicamente para pares nombre-valor, como términos y sus definiciones o metadatos. A diferencia de otros tipos de listas, esta utiliza una relación padre-hijo entre los términos y sus descripciones.

  • <dl>: El contenedor oara toda la lista.
  • <dt>: El término que se está definiendo
  • <dd>: La descripción del término. 
            
              <dl>
                <dt>HTML</dt>
                <dd>HyperText Markup Language</dd>
                <dt>CSS</dt>
                <dd>Cascading Style Sheets</dd>
              </dl>
HTML
HyperText Markup Language
CSS
Cascading Style Sheets

Listas anidadas

Puedes colocar una lista dentro de otra para crear submenús o estructuras jerárquicas. Para garantizar que el anidamiento sea válido, la lista interna debe de estar ubicada dentro de un <li> elemento de la lista principal. 

            
              <ul>
                <li>Coffee</li>
                <li>Tea
                  <ul>
                    <li>Green Tea</li>
                    <li>Black Tea</li>
                  </ul>
                </li>
              </ul>
  • Coffee
  • Tea
    • Green Tea
    • Black Tea

Buenas prácticas para listas HTML

Para mantener la integridad semántica y la accedibilidad, siga estas directrices:

  1. Utilice la semántica correcta: Utilice siempre <ul> para listas con viñetas y <ol> para listas numeradas. Evite usar etiquetas de párrafo o saltos de línea para "simular" una lista.

  2. Evite las listas vacías: Cada <ul>, <ol> lista debe contener al menos un elemento <li> hijo.

  3. Utilice CSS para dar estilo: No utilice atributos HTML para dar estilo a los marcadores de lista. Utilice la list-style-type propiedad CSS para cambiar los estilos de las viñetas o la numeración.

  4. La accesibilidad es primordial: los lectores de pantalla utilizan estas etiquetas para informar a los usuarios cuántos elementos hay en una lista y dónde se encuentran. El uso de listas mejora la navegación para usuarios con discapacidad visual.

  5. Manténgalo limpio: No anide listas a niveles más profundos de lo necesario, ya que un anidamiento excesivo dificulta la lectura y el mantenimiento del código.
