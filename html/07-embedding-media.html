Incrustar contenido multimedia

La inserción de contenido multimedia en HTML consiste en integrar imágenes, audio y video directamente en una página web. Esto permite a los usuarios visualizar o escuchar esos elementos sin necesidad de descargarlos por separado ni navegar a otro sitio. HTML proporciona elementos específicos, como <img>, <audio> y <video>, para facilitar este proceso de inserción, junto con atributos que controlan su aparencia y compartamiento.

Imágenes en HTML

Las imágenes son una forma de mostrar contenido visual en una página web. Se añaden a los documentos HTML mediante la etiqueta <img>, que enlaza con un archivo de imagen. Atributos como src (origen), alt (texto alternativo), width y height se utilizan para específicar la ubicación, la descripción y las dimensiones de la imagen.

Audio

El audio en la web consiste en incorporar archivos de sonido en documentos HTML. Esto permite a los usuarios escuchar música, podcasts u otro contenido de audio directamente en una página web. El elemento <audio> se utiliza para insertar audio, propocionando atributos para controlar la reproducción, mostrar controles y especificar múltiples archivos de origen para garantizar la compatibilidad con los navegadores.

Video

El elemento <video> se utiliza para insertar video. Permite mostrar archivos de video, controlar la reproducción y ofrecer opciones para que los usuarios interactúen con el video, como pausarlo, reproducirlo, ajustar el volumen y verlo en pantalla completa. Este elemento admite varios formatos de video y atributos para personalizar su apariencia y comportamiento.

Iframe

Un marco en Línea iframe (Inline frame) es un elemento HTML que permite insertar otro documento HTML dentro de la página HTML actual. Básicamente, crea una ventana dentro de la página web donde se puede mostrar contenido de una fuente diferente, como otro sitio web, un vídeo o incluso otro archivo HTML alojado en el mismo servidor.

Pilítica de seguridad de contenido

La Política de Seguridad de Contenido (CSP) es un estándar de seguridad diseñado para prevenir ataques de secuencias de comando ente sitios (XSS), clickjacking y otros ataques de inyección de código. Funciona permitiendo definir una lista blanca de fuentes desde las que el navegador tiene permiso para cargar recursos. Esto ayuda a controlar el origen de scripts, hojas de estilos, imagenes y otros recursos, reduciendo así la superficie de ataque de una aplicación web.
